Взять задачу в работу.

1. Посмотри список задач: `ls Tasks/backlog/`
2. Если задача указана явно (`/task 001`) — бери её
3. Если не указана — бери задачу с наименьшим номером (приоритет)
4. Прочитай файл задачи полностью
5. Прочитай `.claude/CLAUDE.md` для контекста архитектуры
6. Перед началом работы выведи план:
   ```
   Задача: [номер] [название]
   План:
   1. ...
   2. ...
   3. ...
   Ожидаемый результат: ...
   ```
7. Выполни задачу по шагам
8. После каждого значимого шага — проверь что бот запускается (если применимо)
9. По завершении — перемести файл: `mv Tasks/backlog/NNN_*.md Tasks/done/`

## Правила

- Следуй стилю кода из CLAUDE.md
- Cog: файл в `Bot/cogs/`, slash-команды с описаниями на русском
- View: файл в `Bot/views/`, persistent views с timeout=None для главного меню
- Task: файл в `Bot/tasks/`, `@tasks.loop(seconds=N)`
- Embed: цвета по статусу (green/red/blue/yellow)
- Проверяй что существующий код не сломан
- Коммить после завершения задачи

## Важно: работа с существующим кодом

Перед переписыванием файла — **прочитай оригинал** из Telegram-версии.
Сохраняй всю бизнес-логику, SQL-запросы, API-вызовы.
Меняй только транспортный слой (отправка сообщений, обработка команд, UI).

## Префиксы коммитов

`[init]` инфраструктура, `[cog]` команда, `[view]` UI-компонент, `[task]` фоновая задача, `[embed]` форматирование, `[config]` конфигурация, `[migration]` Alembic, `[docker]` контейнеризация, `[test]` тесты, `[fix]` баг, `[refactor]` рефакторинг, `[docs]` документация

## Формат коммита

```
[prefix] Краткое описание на русском

- Что сделано
- Какие файлы затронуты
```

Если задача непонятна — не додумывай, спроси.
